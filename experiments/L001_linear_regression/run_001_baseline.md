# run_001_baseline

## 目标

跑通最小线性回归训练闭环。

## 配置

- model: `LinearRegressionModel`
- loss: `MSELoss`
- optimizer: `SGD`
- learning_rate: `0.05`
- epochs: `80`
- dataset: synthetic `y = 2x + 1 + noise`

## 结果

- val loss: `0.039357006549835205`
- learned weight: `1.9962878227233887`
- learned bias: `1.016717791557312`

## 观察

baseline 使用 `lr=0.05`，训练 80 个 epoch 后：

- `weight` 接近真实值 `2.0`。
- `bias` 接近真实值 `1.0`。
- `val_loss` 稳定在约 `0.039`。

学习率对比：

| learning_rate | epoch 80 val_loss | learned weight | learned bias | 观察 |
| --- | ---: | ---: | ---: | --- |
| `0.0005` | `1.3533` | `1.1136` | `0.9148` | 学习率太小，80 个 epoch 内没有充分收敛 |
| `0.005` | `0.0481` | `1.9512` | `1.0215` | 可以收敛，但比 baseline 慢 |
| `0.05` | `0.0394` | `1.9963` | `1.0167` | 收敛稳定，效果最好 |
| `0.5` | `0.0505` | `1.9516` | `0.9989` | 当前数据上仍可训练，但波动更明显 |
| `1.0` | `2222.5691` | `-8.7761` | `-44.6429` | 学习率过大，loss 发散，训练崩掉 |

运行时出现 CUDA driver 过旧的 warning，但当前实验只依赖 CPU 完成最小线性回归训练，不影响本次学习率结论。

## 结论

学习率控制每次参数更新的步子大小。学习率太小会导致收敛很慢；学习率合适时，模型能稳定学到接近真实规律的参数；学习率过大时，即使梯度方向有意义，参数也可能一步走太远，导致 loss 震荡或发散。

当前任务使用 `MSELoss`，是因为线性回归预测的是连续数值，loss 需要衡量预测值和真实值之间的数值差距。`MSELoss` 计算平均平方误差，会更强地惩罚大的预测错误。分类任务则通常使用 `CrossEntropyLoss`，因为分类目标是提高真实类别的预测概率。
