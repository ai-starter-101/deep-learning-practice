# L001: 线性回归与训练闭环

## 学习目标

用最小 PyTorch 例子理解训练闭环：数据、模型、loss、反向传播、参数更新、验证评估。

## 我的当前理解

整个模型训练过程可以理解为：

1. 先准备数据。本例生成 200 条线性回归数据，其中 160 条作为训练集，40 条作为验证集。
2. 定义一个线性模型。模型用当前参数根据输入 `x` 预测 `pred`。
3. 前向传播负责计算预测值和 loss：`pred = model(x)`，`loss = loss_fn(pred, y)`。
4. `optimizer.zero_grad()` 清空旧梯度，避免上一批数据的梯度累积到当前 batch。
5. `loss.backward()` 做反向传播，计算每个参数的梯度，也就是“为了让 loss 变小，参数应该往哪个方向调整”。
6. `optimizer.step()` 根据梯度真正更新参数。
7. 验证集不参与参数更新，只用来检查模型在没见过的数据上的效果。

一句话总结：训练不是模型自动变聪明，而是每一轮先用 loss 衡量预测错多少，再用反向传播计算参数该怎么改，最后由 optimizer 更新参数。

## 实验结果观察

已在远程 Linux 环境 `zj-56.33linux` 跑通最小训练脚本。

学习率实验结果：

- `lr=0.0005`：80 个 epoch 内 `val_loss=1.3533`，明显还没收敛，说明学习率太小会导致训练很慢。
- `lr=0.005`：`val_loss=0.0481`，可以收敛，但速度比 `0.05` 慢。
- `lr=0.05`：`val_loss=0.0394`，`weight=1.9963`，`bias=1.0167`，效果最稳定。
- `lr=0.5`：`val_loss=0.0505`，这组随机条件下还没有崩，但波动更明显。
- `lr=1.0`：`val_loss=2222.5691`，`weight=-8.7761`，`bias=-44.6429`，训练发散。

## 反例 / 边界条件

- 如果忘记 `optimizer.zero_grad()`，梯度会跨 batch 累积，导致更新不再只代表当前 batch，训练过程可能变得不可控。
- 如果学习率过大，梯度方向即使正确，参数也可能一步走太远，导致 loss 震荡、发散，甚至出现 `nan` / `inf`。
- 如果验证集也参与 `optimizer.step()`，就相当于模型提前“看过考试题”，验证结果会失去评估意义。
- 当前例子没有单独划分测试集，因为 L001 的目标是先理解最小训练闭环；后续学习过拟合和模型评估时再引入 test set。

## 我已经验证过什么

今日通过代码阅读和问答验证了以下理解：

- `train_set` 用来更新参数，`val_set` 用来评估没参与训练的数据表现。
- 前向传播负责计算预测值和 loss。
- 反向传播和参数更新是两个步骤：`loss.backward()` 计算梯度，`optimizer.step()` 更新参数。
- 梯度裁剪的 `max_norm=1.0` 限制的是所有参数梯度合起来的整体范数，而不是每个单独梯度都不能超过 1。
- 通过学习率实验验证了：学习率太小会收敛慢，学习率太大可能导致 loss 发散。
